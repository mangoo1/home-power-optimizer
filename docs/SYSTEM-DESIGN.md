# Home Power Optimizer — 系统设计文档

> 最后更新：2026-04-25

---

## 一、系统目标

用最低成本保证全天用电，同时：
1. **太阳能零浪费** — PV 全部自消纳，不低价卖出
2. **热水器每天有热水** — 自动安排最优时段开关
3. **晚间高价卖电** — PV 充满后多余电量在高价时卖出获益
4. **过夜电量安全** — 21:00 保留足够电量撑过夜间消耗

---

## 二、硬件约束

| 参数 | 值 | 说明 |
|------|-----|------|
| 电池容量 | 42 kWh | ESS 系统 |
| 逆变器限流 SOC | 93% | 超过此值 PV 无法充入电池 |
| 主断路器 | 7.7 kW | 从电网取电总上限 |
| 安全 buffer | 0.5 kW | 断路器余量保留 |
| 最大充电功率 | 5 kW | 逆变器硬限制 |
| 最大放电功率 | 5 kW | 逆变器硬限制 |
| PV 系统峰值 | 4.2 kW | 实测（秋季约 2kW 峰值） |
| PV 换算系数 | 0.0032 kW per W/m² | 97样本实测均值 |
| 主热水器 | ~5 kW | Main，bf160bbe78f4f1ce6dpkdp |
| GF 热水器 | ~3.5 kW | GF，bf3c28e8181e5e980eoobm |

**充电功率实时计算公式：**
```
maxChargeKw = min(5, BREAKER - BUFFER - max(0, homeLoad - pvPower))
```
- PV 已计入（抵消 homeLoad），PV 出力越大可充越多
- 热水器开着时：homeLoad = 家用 + 5kW，maxChargeKw 自动降低
- 断路器保护统一处理所有情况，不需要手动干预

## 三、每日计划逻辑（plan-today.js）

### 3.1 整体流程

```
1. 拉取 Open-Meteo 太阳辐射预测 → 计算逐小时 PV 估算
2. 拉取 Amber API 电价预测（用 startTime UTC→Sydney，非 nemTime）
3. 计算热水器窗口（主 + GF）
4. 生成充放电计划（Phase 1: 电网充电槽选取，Phase 3: 逐槽决策）
5. 写入逆变器时间窗口
6. 存入 daily_plan 表，发送 WhatsApp 报告
```

### 3.2 电网充电策略

**目标：电网充到 85% SOC，给 PV 留 8% 缓冲空间（85%→93%）**

- 贪心算法选今天最便宜的槽，充够 `85% - 当前SOC` 所需的 kWh
- SOC < 65% 时，早晨所有低价槽都纳入充电（不等最便宜的）
- 充电截止时间：高电价/DW 起点前，最晚 17:00

### 3.3 PV 消纳策略

**原则：有 PV 出力 + 电池 < 93% → 始终充电**

```
if hasPv && soc < 93%:
    chargeKw = maxChargeKw  // 用满断路器余量
```

- 热水器开着时：热水器消纳部分 PV，充电消纳剩余断路器余量
- 热水器关了后：继续充电消纳 PV 尾端，直到93%
- **PV 时段禁止卖电**（pvKw > 0.2 时不卖）

### 3.4 卖电策略

- 仅在 **PV 落山后**（pvKw < 0.2）且 h >= 16 时卖电
- feedIn 门槛 = max(SELL_FLOOR_C, 买入均价 + SELL_MIN_MARGIN_C)
- 卖电保底 SOC = SOC_SELL_FLOOR（默认 50%），确保过夜够用

## 四、热水器计划逻辑

### 4.1 设计原则

- **两台热水器不能同时开**
- **热水器排在 PV 峰值时段**，让 PV 直接供热水器，天然消纳
- **所有热水器必须在危险截止前结束**，避免 DW 或高电价
- **越早开越好**：热水器结束后剩余 PV 时段更多，电池有更多时间充满

### 4.2 时间窗口计算

```
危险截止 = 第一个 buyC > HW_GRID_MAX_C(10¢) 或 DW 的时段（14:00后）
主热水器截止 = 危险起点 - 30min - 60min（留1小时给GF）
GF截止 = 危险起点 - 30min

主热水器：在主热水器截止前，找最早的 avgPV >= 峰值60% 的连续4槽（2小时）
GF热水器：主热水器结束后，GF截止前，找最早的 avgPV >= 峰值60% 的连续2槽（1小时）
```

**有 DW 示例（DW 15:00 开始）：**
- 危险截止 = 15:00
- 主热水器截止 = 13:30 → 主热水器 11:30–13:30
- GF截止 = 14:30 → GF 13:30–14:30

### 4.3 供电模式

- buyC ≤ 10¢ → **电网直供**（断路器保护自动限制充电功率）
- buyC ≥ 12¢ → **电池供电**（Self-use 模式）

### 4.4 GF 凌晨固定窗口

- **04:00–05:00 固定开**（电池供热，不受电价影响）
- 由 plan-executor 兜底执行，plan-today 不管凌晨时段

## 五、执行器逻辑（plan-executor.js）

**每 5 分钟执行一次，职责：**
1. 读取 ESS 实时状态 + Amber 实时电价
2. 对照 daily_plan 当前时段，动态调整充电功率
3. 检查热水器窗口，到点开关
4. 断路器保护（gridImport 超限时自动降功率）
5. Turso 云同步

**充电功率原则：**
- 充电槽：`targetKw = calcSafeChargeKw(homeLoad, pvPower)`（实时最大值）
- 不受 plan 静态 chargeKw 限制，每5分钟重算

**热水器触发窗口：**
- 窗口开始后 30 分钟内触发（不是 ±5 分钟），避免 plan 重生成后错过

## 六、关键 Bug 修复记录

### Amber 时区偏移（2026-04-25）
**症状**：所有电价时段偏移 30 分钟  
**原因**：代码用 `nemTime`（NEM 结算时间，比实际开始晚30分钟）聚合价格槽  
**修复**：改用 `startTime`（UTC 转 Sydney 时间）

### 热水器错误触发（2026-04-25）
**症状**：Heartbeat 09:31 自动开热水器，与 plan 计划冲突  
**原因**：HEARTBEAT.md 残留旧热水器逻辑，AI 主 session 自行执行  
**修复**：HEARTBEAT.md 加硬性禁令，热水器控制完全移入 plan-executor

### 充电功率被截断（2026-04-25）
**症状**：热水器开着时充电只有 1.67kW，应为 3.6kW  
**原因**：ess-api.js 的 BREAKER_BUFFER 硬编码 0.3，与 .env 的 0.5 不一致  
**修复**：ess-api.js 改从 .env 读取，统一为 0.5

## 七、配置参数（.env）

所有阈值在 `.env` 文件里，修改无需改代码：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| CHARGE_TARGET_PCT | 85 | 电网充电目标 SOC % |
| SOC_PV_LIMIT_PCT | 93 | 逆变器限流 SOC %，PV 无法充入 |
| SOC_SELL_FLOOR_PCT | 50 | 卖电保守底线 % |
| SOC_MIN_PCT | 10 | 电池绝对底线 % |
| BUY_MAX_C | 14 | 买电上限 ¢/kWh |
| SELL_FLOOR_C | 10 | 卖电绝对下限 ¢/kWh |
| SELL_MIN_MARGIN_C | 3 | 卖电最低利润 ¢/kWh |
| HW_GRID_MAX_C | 10 | 热水器电网直供门槛 ¢ |
| HW_BATT_MIN_C | 12 | 热水器电池供电门槛 ¢ |
| BREAKER_KW | 7.7 | 主断路器 kW |
| CHARGE_BUFFER_KW | 0.5 | 断路器安全余量 kW |
| PV_SCALE | 0.0032 | PV 换算系数 kW per W/m² |
| PV_PEAK_KW | 4.2 | PV 实测峰值 kW |

## 九、架构原则

> 所有太阳能、电池、热水器相关逻辑必须保持独立，目标是迁移到独立机器运行。

- 逻辑放在 `home-power-optimizer/` 独立脚本里
- Heartbeat 只做监控提醒，不做任何控制
- 所有控制由 plan-executor 统一执行
- 迁移到新机器只需复制目录 + 配置 .env
